# PROJECT_RULES.md
> 프로젝트 생성 시 복사해서 사용. 항목별로 프로젝트에 맞게 수정.

## 1. 디버그 로그 원칙
- **모든 외부 호출(LLM, API, DB)의 입력과 출력을 로그로 남긴다**
- 입력: 첫 200자 + 전체 길이
- 출력: 첫 300자 + 성공/실패 여부
- 에러: 전체 에러 메시지 + 어떤 입력에서 발생했는지
- 로그 포맷: `[함수명] key=value` (grep으로 찾을 수 있게)

## 2. 검증 단계 의무화
- LLM 출력은 반드시 1회 이상 검증 후 다음 단계로 전달
- 검증 실패 시: 1회 자동 재시도 → 실패하면 에러 로그 + 상태 변경 (무한 재시도 금지)
- 검증 로직은 LLM 없이 코드로 (비용 0)

## 3. 타임아웃 필수
- 모든 외부 API 호출에 timeout 설정 (기본 30초)
- DB 쿼리: 10초
- 크롤링: 10초
- LLM 호출: 30초
- 타임아웃 발생 시: catch에서 어떤 호출이 타임아웃됐는지 로그

## 4. 입력→출력 추적 가능해야 함
- "결과가 이상할 때, 입력이 뭐였는지 5초 안에 확인 가능"이 기준
- 로그만 보고 "입력이 잘못됐는지 / 처리가 잘못됐는지 / 출력이 잘못됐는지" 구분 가능해야 함

## 5. dryRun 모드
- 비용이 드는 파이프라인은 dryRun 모드 제공
- dryRun: 입력 파싱 + 중간 변환까지만 실행, 실제 API 호출 안 함
- 목적: 배포 전 입력 품질 확인 (크레딧/비용 안 쓰고)

## 6. 상태 머신 명확화
- 모든 비동기 파이프라인은 상태(status)를 DB/로그에 기록
- 상태 전이: pending → processing → completed/failed
- "지금 어디서 멈춰있는지" 외부에서 조회 가능해야 함
- 상태가 10분 이상 안 바뀌면 문제로 간주

## 7. 실패 후 학습
- 버그 발견 → 원인 1줄 + 방지책 1줄을 이 파일 하단에 추가
- 형식: `[날짜] 증상 → 원인 → 방지책`
